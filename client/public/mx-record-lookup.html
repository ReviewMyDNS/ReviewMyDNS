MX Record Lookup

Find and verify Mail Exchange (MX) records for any domain to ensure proper email delivery.

How to Read MX Records

An MX record tells email servers where to send mail for a domain. It has two main parts:

  • Hostname/Value: The address of the mail server (e.g., aspmx.l.google.com).
  • Priority: A number that indicates the order in which servers should be used. Mail is sent to the server with the lowest priority number first. If that server isn't available, the next lowest is tried.

Common MX Record Mistakes

Pointing to an IP Address

MX records must always point to a hostname, never directly to an IP address. If your mail server is at a specific IP, you must create an A record pointing a hostname to that IP, and then point your MX record to that hostname.

Pointing to a CNAME

The DNS specification (RFC 1034, RFC 2181) forbids pointing an MX record to a CNAME alias. This can cause unexpected behavior and delivery failures. Always use an A record.

Incorrect Priority Numbers

Using the same priority for primary and backup mail servers can lead to unpredictable mail flow. Ensure your primary server has the lowest priority number (e.g., 10) and backup servers have higher numbers (e.g., 20, 30).

MX Records for Popular Email Providers

Google Workspace

If you use Google Workspace for email, your MX records should be:

  • ASPMX.L.GOOGLE.COM (Priority: 1)
  • ALT1.ASPMX.L.GOOGLE.COM (Priority: 5)
  • ALT2.ASPMX.L.GOOGLE.COM (Priority: 5)
  • ALT3.ASPMX.L.GOOGLE.COM (Priority: 10)
  • ALT4.ASPMX.L.GOOGLE.COM (Priority: 10)

Microsoft 365 (Office 365)

Microsoft 365 uses a unique MX record format based on your domain. You can find the exact value in your Microsoft 365 admin center. It will look like this:

  • your-domain-key.mail.protection.outlook.com (Priority: 0)

Frequently Asked Questions (FAQ)

How many MX records can a domain have?

A domain can have multiple MX records. This is a common practice to provide redundancy. If the primary mail server (lowest priority) is down, sending servers will try the next one in the list.

What does the priority number mean?

The priority number dictates the order of preference. A lower number means a higher priority. Mail will always be delivered to the available mail server with the lowest priority number first.
